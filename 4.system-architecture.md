# System Architecture

## 1. High-Level Architecture
- **Client/Frontend Layer:** Next.js application, server-side rendered (SSR), hosted on Vercel for fast global delivery.
- **API/Backend Layer:** Python FastAPI, hosted on Render or Railway.
- **Database Layer:** PostgreSQL (Supabase or Neon) for structured data storage.
- **Asset Delivery:** CDN (Cloudinary) for serving images and logos.

## 2. Real-Time Pipeline
- **WebSockets:** FastAPI handles WebSocket connections (`/ws/market-data/{ticker}`) to stream live price ticks directly to the Next.js frontend.
- **Frontend Syncing:** The Next.js frontend connects to the WebSocket upon accessing specific stock pages and updates the Framer Motion-enabled charts without long-polling.

## 3. UI and State Architecture
- **State Management:** Zustand or Redux Toolkit manages global state: Session info, Theme preference, Dashboard layout config.
- **Styling & Animation:** Tailwind CSS and Framer Motion.
- **Charting Engine:** Lightweight Charts or Recharts for performant rendering of high-volume financial data.
