# Database Schema

## 1. Overview
The database uses PostgreSQL (hosted on Supabase or Neon). The schema is designed to be relational, reliable, and capable of handling user data, blog content, tool information, and market data.

## 2. Tables

### Users Table
- `id`: UUID, Primary Key
- `email`: VARCHAR, Unique
- `password_hash`: VARCHAR
- `theme_preference`: ENUM ('dark', 'light'), Default 'dark'
- `dashboard_layout_config`: JSONB (Stores bento-box grid layout arrangements)
- `created_at`: TIMESTAMP

### Blogs Table
- `id`: UUID, Primary Key
- `title`: VARCHAR
- `content_markdown`: TEXT
- `author`: VARCHAR
- `read_time_minutes`: INTEGER
- `hero_image_url`: VARCHAR
- `tags`: TEXT[] (Array of tags)
- `published_at`: TIMESTAMP

### AITools Table
- `id`: UUID, Primary Key
- `name`: VARCHAR
- `category`: VARCHAR
- `pros`: TEXT[] (Array of pros)
- `cons`: TEXT[] (Array of cons)
- `pricing_tier`: VARCHAR
- `logo_url`: VARCHAR

### Stocks Table
- `ticker`: VARCHAR, Primary Key
- `company_name`: VARCHAR
- `exchange`: VARCHAR (e.g., NASDAQ, NSE)
- `asset_type`: ENUM ('Stock', 'Index')
