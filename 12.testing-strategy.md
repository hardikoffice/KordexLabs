# Testing Strategy

## 1. Unit Testing
- **Frontend:** Jest and React Testing Library for testing atomic UI components (e.g., button rendering, tag styling) and Zustand state logic.
- **Backend:** Pytest for robust testing of FastAPI endpoints, database CRUD operations, and data parsing logic.

## 2. Integration Testing
- Verify the communication between the Next.js application and FastAPI endpoints.
- Ensure WebSocket connections can be established, maintain stability, and properly handle payloads.

## 3. End-to-End (E2E) Testing
- Use Playwright or Cypress to simulate core user flows:
  - User logging in and customizing their bento-box dashboard.
  - Navigating to the Tools Comparison Matrix and verifying horizontal scroll/data load.
  - Viewing a stock chart and ensuring real-time UI updates occur without errors.

## 4. UI/UX Verification
- Manual testing of dark/light mode toggles to ensure high contrast and readability.
- Visual regression testing for responsive design across Desktop, Tablet, and Mobile viewports.
